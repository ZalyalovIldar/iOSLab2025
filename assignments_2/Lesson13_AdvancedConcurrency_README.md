# 📘 Lesson 12 — Advanced Concurrency

## 🎯 Цель
Познакомиться с продвинутой работой с конкурентными задачами в Swift и научиться защищать данные от race conditions.

Темы урока:

- `Task`
- `async / await`
- `TaskGroup`
- `actor`
- `@MainActor`
- `Task cancellation`

---

# 📱 Задание — Actor‑based Cache

Добавьте **кэширование данных пользователей** с использованием `actor`, чтобы предотвратить повторные сетевые запросы.
  
Сделайте приложение для подгрузки пользователей из любого открытого API.
Используйте хотя бы 1 раз TaskGroup. 
Ваша задача — добавить **потокобезопасный кэш**.

---

## 📌 Требования

Создать:

```
actor UsersCache
```

Кэш должен:

- хранить пользователей в памяти
- предотвращать повторные сетевые запросы
- обеспечивать thread‑safe доступ

Пример интерфейса:

```swift
func user(for id: Int) -> User?
func save(_ user: User)
```

---

## 💻 Уровни выполнения

### Base

- реализован `actor` для кэша
- данные сохраняются после загрузки
- повторный запрос использует кэш

### Plus

- реализована отмена задач (`Task.cancel()`)
- предотвращены дублирующиеся запросы

### Pro

- ограничение количества параллельных запросов
- логирование обращений к кэшу

---

## ✅ Критерии приёмки

- данные загружаются корректно
- повторные запросы используют кэш
- отсутствуют race conditions
- приложение компилируется и работает без ошибок
